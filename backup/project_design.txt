## 一、测试分层设计思路

基于你的已有框架（`pages/`、`base_page.py`、`conftest.py`、`pytest.ini`），登录后的测试用例设计核心思路是：

> **横向拆分功能模块，纵向遵循依赖链（登录 → 商品 → 购物车 → 结算）**

SauceDemo 登录后的核心业务流程可以概括为以下几个测试模块：

| 模块 | 测试页面 | 依赖关系 |
|------|---------|---------|
| **Products** | `/inventory.html` | 登录后默认到达 |
| **Product Detail** | `/inventory-item.html?id=n` | 从商品列表点击进入 |
| **Cart** | `/cart.html` | 需要先有商品添加操作 |
| **Checkout**（含 Info + Overview + Complete） | `/checkout-step-one.html` → `/checkout-step-two.html` → `/checkout-complete.html` | 需要先有购物车数据 |

### 目录职责说明

- **`base/base_page.py`**：封装 `find_element`、`wait_for_element`、`click`、`input_text`、`get_text` 等通用方法，所有页面类继承它。
- **`common/test_data.py`**：存放测试使用的常量数据，如登录凭证、商品名称、用户信息等。
- **`pages/`**：每个页面独立的类，封装该页面特有的元素定位和业务操作方法，禁止在测试用例中直接写元素定位表达式。
- **`testcases/`**：按功能模块拆分测试文件，测试用例中只调用页面对象的方法，不写具体定位逻辑，不包含硬编码数据。

## 四、测试用例设计（按模块详述）

### 1. `test_products.py` — 商品模块测试（约 8~10 条用例）

#### 1.1 页面加载与商品展示

- `test_products_page_title_displayed`：登录后商品页标题是否为"Products"
- `test_product_list_visible`：验证商品列表可见，至少有 6 个商品
- `test_each_product_displays_required_fields`：验证每个商品展示了图片、标题、价格和 Add to Cart 按钮

#### 1.2 商品排序功能（SauceDemo 右上角下拉菜单支持排序）

- `test_sort_by_name_az`：按名称 A → Z 排序验证
- `test_sort_by_name_za`：按名称 Z → A 排序验证
- `test_sort_by_price_low_to_high`：按价格低→高排序验证
- `test_sort_by_price_high_to_low`：按价格高→低排序验证

#### 1.3 添加/移除商品

- `test_add_one_product_to_cart`：添加一个商品后，购物车角标数量变为 1，按钮文字变为 Remove
- `test_add_multiple_products_to_cart`：添加多个商品，购物车角标计数正确累加
- `test_add_and_remove_product_updates_cart_badge`：添加后再移除，购物车角标应消失或归零
- `test_add_same_product_twice`：同一商品是否可重复添加并计数正确（验证界面逻辑）

#### 1.4 商品详情页跳转

- `test_click_product_title_opens_detail_page`：点击商品标题跳转到正确的商品详情页（URL 包含 `id` 参数，页面展示该商品信息）

---

### 2. `test_cart.py` — 购物车模块测试（约 6~8 条用例）

> ⚠️ 此模块依赖 `test_products.py` 中的添加商品操作，建议使用 `conftest.py` 中的 fixture 复用登录态。

#### 2.1 购物车内容展示

- `test_cart_displays_all_added_items`：添加两件商品后进入购物车，验证购物车内商品名称与实际添加一致
- `test_cart_item_prices_match_inventory`：验证购物车中价格与商品页价格一致
- `test_cart_item_quantity_display`：SauceDemo 购物车中每个商品显示数量为 1（数量固定，可验证）

#### 2.2 购物车操作

- `test_remove_item_from_cart_on_cart_page`：在购物车页面移除商品，验证该商品从购物车消失
- `test_continue_shopping_returns_to_inventory`：点击 Continue Shopping 返回商品页，验证 URL 或页面标题
- `test_empty_cart_shows_empty_state`：清空购物车后验证空状态信息或角标消失

#### 2.3 跨页面同步验证

- `test_cart_badge_count_syncs_with_cart_items`：添加/移除商品时，商品页角标与购物车内商品数量同步更新
- `test_cart_persistence_after_logout`：添加商品后登出再登入，验证购物车为空（可选，验证 Session 行为）

---

### 3. `test_checkout.py` — 结算流程测试（约 8~10 条用例）

> 建议使用 fixture 预先添加 1~2 件商品到购物车，以复用登录和添加商品的准备步骤。

#### 3.1 结算信息验证

- `test_checkout_form_validation_all_fields_required`：全为空，点击 Continue 应提示错误信息
- `test_checkout_form_validation_missing_first_name`：仅缺少 First Name 时触发错误
- `test_checkout_form_validation_missing_last_name`：仅缺少 Last Name 时触发错误
- `test_checkout_form_validation_missing_postal_code`：仅缺少 Postal Code 时触发错误
- `test_valid_form_proceeds_to_overview`：填写有效信息后点击 Continue 进入订单概览页

#### 3.2 订单概览验证

- `test_overview_displays_all_cart_items`：概览页展示的商品名称与购物车内容一致
- `test_overview_item_total_calculation`：验证商品小计价格与购物车内价格总和一致
- `test_overview_total_price_with_tax`：验证最终总价 = 商品小计 + 税费
- `test_cancel_button_overview_returns_to_inventory`：概览页点 Cancel 返回商品页

#### 3.3 订单完成验证

- `test_checkout_completion_flow`：完整结算流程验证，完成后显示成功信息"Thank you for your order！"或类似措辞
- `test_checkout_completion_redirects_to_inventory_after_complete`：Finish 后回到商品页，购物车清空

## 六、关键技术建议

✅ **推荐在 `conftest.py` 中创建 fixture 优化登录**

```python
import pytest
from pages.login_page import LoginPage

@pytest.fixture
def logged_in_driver(driver):
    """登录后的driver fixture，供购物车和结算测试复用"""
    login_page = LoginPage(driver)
    login_page.navigate_to()  # 打开登录页
    inventory_page = login_page.login("standard_user", "secret_sauce")
    yield driver
    # 可选 teardown
```

## 七、后续扩展建议

以上设计已覆盖 SauceDemo 的核心业务链路。待这些用例稳定运行后，建议进一步覆盖以下场景：

| 扩展方向 | 说明 |
|---------|------|
| **异常用户测试** | 分别使用 `locked_out_user`、`problem_user`、`performance_glitch_user` 验证不同用户的页面行为差异 |
| **UI样式/视觉回归** | 添加 snapshots 比对，确保前端改动不影响页面关键元素的展示 |
| **并发与稳定性** | 在 CI 中配置定时执行，监控每个用例的稳定性和执行耗时 |
| **数据驱动测试** | 使用 pytest 的 `@pytest.mark.parametrize` 对多个商品进行遍历测试，避免硬编码重复 |
